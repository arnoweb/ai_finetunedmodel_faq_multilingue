03

Model registry

There is no local model server — every app calls SentenceTransformer(path) directly, which resolves either to a public Hub id or the fine-tuned repo, and caches the loaded model in-process (@st.cache_resource) for the life of the Streamlit session.

sentence-transformers/paraphrase-multilingual-mpnet-base-v2

Public, unmodified — the exact checkpoint the fine-tuned model started from. The only fair "before fine-tuning" reference (verified same architecture: XLMRobertaModel, hidden_size 768, 12 layers).

arnoweb/model-faq-sentence-autotrain

Private fine-tuned repo. Requires a Hugging Face token (env var or Streamlit secret).

04

Serving layer

Two Streamlit apps share the same retrieval mechanics — encode the FAQ answers once, encode the query, rank by cosine similarity — but differ in scope.

AppModels loadedOutput
faq_streamlit.py fine-tuned only live search-as-you-type (streamlit-searchbox), similarity threshold 0.1, optional LLM-generated answer
faq_compare_streamlit.py pre-fine-tuning + fine-tuned side-by-side top-K (user-adjustable), no generation step
evaluate_models_compare.py pre-fine-tuning + fine-tuned MRR, Recall@1/3/5, mean true similarity — printed to stdout, overall & per language
05

Generation (optional)

Only faq_streamlit.py generates prose. build_rag_prompt() assembles the top-3 retrieved Q/A pairs plus the user's question, in the selected UI language, with an explicit instruction to answer "I don't know" / "Je ne sais pas" rather than invent an answer — the app then catches that phrase and shows a warning instead of the raw text.

query_local_llm(prompt, model)

POSTs to localhost:1234/v1/chat/completions. Default model qwen/qwen3-vl-4b. No API key required.

query_remote_llm(prompt, model)

POSTs to the HF router with Bearer HF_API_KEY (from .env). Raises if the key is missing.

06

Request flow — faq_streamlit.py

What happens between a keystroke and an answer on screen.

  1. Language & query

    User picks English/Français and types a question; this selects faq_data_paths[language].

  2. Model & corpus load (cached)

    load_model() loads the fine-tuned SentenceTransformer once; load_faq_data() and compute_answer_embeddings() cache per language so repeat queries skip re-encoding.

  3. Similarity search

    The query is embedded and compared to all answer embeddings with util.pytorch_cos_sim; the top 3 indices are taken.

  4. Confidence gate

    If the best score is below SIMILARITY_THRESHOLD = 0.1, the app shows a "no relevant answer" message instead of results.

  5. Results rendered

    The 3 matched Q/A pairs are shown ranked, each with its similarity score.

  6. Optional generation

    On button click, build_rag_prompt() packages the 3 results + question; query_remote_llm() returns prose grounded in them, or a caught refusal is shown as a warning.

08

Environments & deployment

Dev container

.devcontainer/devcontainer.json — Python 3.11 image, installs requirements.txt on create, auto-runs faq_streamlit.py on attach, forwards port 8501.

Local ports

Main RAG app on :8501, comparison app on :8502 — run side by side for A/B testing.

Streamlit runtime

.streamlit/config.toml holds server/runtime settings shared by both apps.

Training vs serving split

Fine-tuning runs locally against models/config/config.yml; only the resulting model is pushed to the Hub, so serving never needs the training environment.
